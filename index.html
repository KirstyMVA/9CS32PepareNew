Week 34 Develop: Nested Selection Game Logic

The Purpose of the Algorithm

When developing video games, computers must constantly evaluate live events to update a player's core stats. This algorithm is designed to manage a penalty shootout scenario. It handles setting up a player's session values, rolling a random event to see if the penalty was saved or missed, dynamically updating their metrics, and terminating the game instantly if they run completely out of lives.

Your Instructions

You need to reconstruct this game algorithm using Pseudocode blocks. Arrange the shuffled statements on the left into the empty sequence tracks on the right so they precisely follow this step-by-step logic path:

  1. Set Up: Establish the starting constraints (lives = 3 and score = 0).
  2. The Event: Randomly simulate an outcome state as either "Scored" or "Missed".
  3. The Branch Check: Use an IF / ELSE statement structure. If they scored, award them 10 points. If they missed, deduct 1 life. Don't forget to close this decision path using an ENDIF!
  4. Elimination Guard: Immediately check if their lives counter has dropped to 0, and display "Game Over" if true.

🕹️ TASK: Click the shuffled blocks on the left to stack them onto the right-hand board. If you make a mistake, simply tap any slot on the right to remove that block.

Pseudocode Blocks

SET score TO 0
SET lives TO 3
SET penalty_result TO RANDOM("Scored", "Missed")
IF penalty_result == "Scored" THEN
SET score TO score + 10
ELSE
SET lives TO lives - 1
ENDIF
IF lives == 0 THEN
OUTPUT "Game Over"
ENDIF

2. CHOOSE THE RANDOM EVENT
3. THE PENALTY OUTCOME (IF / ELSE)
4. CHECK FOR ELIMINATION
