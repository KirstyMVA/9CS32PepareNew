Week 34 Develop: Game Algorithm Planning

⏱️ TIME LIMIT: 30 Minutes. Practice planning an algorithm using Pseudocode statements. Your program needs to handle setting up a game player's status, tracking a penalty, and checking if they have lost.

The Goal

You need to arrange the Pseudocode blocks into the correct sequential order on the right. The algorithm must follow this precise checklist design path:

  1. Create the starting variables for score and lives.
  2. Reduce the player's lives counter by 1.
  3. Check IF the player has run out of lives entirely.
  4. Output a "Game Over" alert window if that condition is met.
