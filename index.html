Week 34 Develop: Nested Logic (Penalty Decider)

⏱️ TIME LIMIT: 30 Minutes. Let's make our algorithm smarter. Instead of automatically losing a life, we will use a random generator to decide if the penalty is scored or missed!

The Logic Rules

Your algorithm needs to use Selection blocks to evaluate a random variable called penalty_result. Arrange the blocks to follow this design path:

  1. Set up the starting game values (lives = 3 and score = 0).
  2. Generate a random status: "Scored" or "Missed".
  3. Check IF the result is "Scored" ➡️ Increase the score by 10.
  4. ELSE (if they missed) ➡️ Take away 1 life.
  5. Finally, check if their lives counter has dropped to 0 to trigger a "Game Over".

🕹️ INSTRUCTION: Click blocks on the left to stack them in order on the right. Tap a slot on the right if you need to remove a block and try again.

Pseudocode Blocks

SET score TO 0
SET lives TO 3
SET penalty_result TO RANDOM("Scored", "Missed")
IF penalty_result == "Scored" THEN
SET score TO score + 10
ELSE
SET lives TO lives - 1
ENDIF
IF lives == 0 THEN
OUTPUT "Game Over"
ENDIF

Your Algorithmic Sequence

1. CHOOSE START VALUES
2. CHOOSE THE RANDOM EVENT
3. THE PENALTY OUTCOME (IF / ELSE)
4. CHECK FOR ELIMINATION
